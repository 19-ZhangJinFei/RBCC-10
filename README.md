# 豆韵 DouYun

豆韵是一个面向中华传统文化主题的拼豆图纸生成网站。项目尝试把传统纹样、民俗意象、文学典故、器物审美和当下流行的手工拼豆创作结合起来，让用户可以从文化主题出发，通过 AI 生成或上传图片，再转化为可实际制作的拼豆图纸、材料清单和制作方案。

项目定位不是简单的“图片转像素”工具，而是一套完整的传统文化拼豆创作工作流：选题、生成、识别、再创作、像素化、配色映射、用量统计、保存、分享和导出。

## 设计理念

拼豆是一种低门槛、强参与感、适合亲子、社群、研学和文创场景的手工形式。传统文化纹样则具有高辨识度、强符号性和丰富的审美来源。豆韵的设计目标是把两者连接起来：

- **传统文化现代化表达**：把青花瓷、敦煌、京剧脸谱、山海经、二十四节气、故宫纹样、四大名著等文化资源转化为适合拼豆颗粒表达的图案。
- **AI 辅助而非替代手作**：AI 负责生成视觉素材、理解主体、组织文化说明；用户仍然可以调整主题、主体识别、提示词、颜色、网格和最终图纸。
- **从屏幕到实物**：所有生成结果都围绕真实制作落地，包括颜色数量限制、色号标注、材料用量、熨烫步骤、导出资料等。
- **面向教学和共创**：内置主题、示例、论坛和帮助文档适合课堂展示、社群工作坊和个人作品沉淀。

## 页面组成与功能

### 首页

首页是整个网站的入口和视觉展示页。

- 顶部导航包含首页、创作、项目、豆韵 AI、论坛、帮助和用户入口。
- 首屏展示品牌标语和快速开始按钮。
- 滚动参考图展示 `public/showcase` 中的传统文化拼豆底稿。
- 制作流程区用四个步骤说明从文化主题到拼豆底稿的过程。
- 论坛入口以四张参考图横向铺满展示，引导进入作品分享页面。
- 帮助入口提供快速跳转到操作指南和常见问题。

### 创作页

创作页是核心工作区，由四个步骤组成。

#### 第一步：配置

用户在这里确定作品的基础参数。

- 选择传统文化主题。
- 选择核心元素和文化叙述。
- 选择作品形式，包括杯垫、挂件、冰箱贴、胸针、吊饰、随身牌等。
- 设置画面比例、网格尺寸、颜色上限。
- 控制是否显示网格、是否平滑杂点、是否连接孤立色块。
- 指定可用颜色，系统会优先把最终图纸映射到这些颜色。
- 支持 AI 生成图案、使用内置样例或上传本地图片。

内置主题覆盖敦煌文化、青花瓷、京剧脸谱、山海经、二十四节气、甲骨文、三星堆、宋代瓷器、汉服纹样、苗绣、壮锦、剪纸、皮影戏、景泰蓝、商周青铜器、故宫宫廷纹样、苏州园林、水墨山水、传统瑞兽、十二生肖、年画与春节、端午文化、中秋文化、中国茶文化、红楼梦、西游记、三国演义、水浒传、诗词书法等方向。

#### 第二步：主体提取与 AI 再创作

第二步处理原始素材和文化风格再创作。

- 上传图片后，默认把整张原图作为主体模板，避免主体未被选中导致无法识别。
- 用户仍可以用选择、增加、减少、框选等工具修改主体区域。
- 系统统计主体颜色占比，作为后续再创作的颜色参考。
- AI 识别主体会返回主体名称、类别、视觉证据、置信度、备选项和视觉摘要。
- 识别结果可编辑，用户可以修正模型误判。
- AI 再创作使用结构化主体识别信息和颜色组成，不再盲目依赖原图文件名或配置页主题。
- 当前 AI 提示词会显示为可编辑文本框，用户可以修改后重新生成。
- 再创作结果会自动识别图案边缘以内的区域作为主体，用于后续拼豆图纸生成。

#### 第三步：拼豆图纸

第三步把再创作结果转成可制作的拼豆图纸。

- 将图像按网格尺寸像素化。
- 使用开源色表映射到最近可用颜色。
- 按颜色数量上限合并相近颜色。
- 对外部浅色背景做连通区域识别并标记为非主体区域。
- 可连接孤立色块，减少实际摆豆时的零散杂点。
- 支持点击颜色高亮同色格，便于对照摆放。
- 显示颜色用量统计和材料占比。
- 支持手动编辑局部颜色。

#### 第四步：制作方案

第四步输出最终制作资料。

- 展示作品标题、拼豆图纸、无标注图和材料清单。
- AI 生成文化说明，包括作品名称、文化来源、图案寓意和设计说明。
- 文化说明同样读取图像和主体识别结果，尽量避免套用配置主题。
- AI 文化文案提示词可编辑，用户修改后可重新生成。
- 输出制作步骤、工具准备、拼豆步骤、熨烫建议、时间和成本估算。
- 支持下载带网格图纸 PNG、无标注 PNG、材料 CSV 和制作方案文本。

### 项目页

项目页用于管理本地作品。

- 自动保存和手动保存创作记录。
- 支持搜索项目标题、主题、元素和作品形式。
- 支持继续编辑、删除和新建项目。
- 项目数据使用浏览器本地存储和 IndexedDB 保存。
- 登录用户和匿名用户的项目记录隔离存储。

### 豆韵 AI 页面

豆韵 AI 是独立的图像生成入口。

- 用户输入生图提示词。
- 服务端调用 Ark 图像生成接口。
- 生成结果以聊天消息形式展示。
- 历史消息保存在浏览器本地。
- 适合先探索视觉灵感，再回到创作流程上传或继续处理。

### 论坛页

论坛页用于作品分享和模板导入。

- 展示内置模板和云端社区作品。
- 支持按标题、主题、作者和元素搜索。
- 用户可以发布当前作品到论坛。
- 点击作品可查看大图、主题、元素、文化说明和推荐配色。
- 支持一键导入论坛作品为自己的可编辑项目。

### 帮助页

帮助页提供完整说明。

- 设计目的。
- 操作指南。
- 常见问题。
- 使用技巧。
- 中英文内容切换。
- 支持侧边目录和移动端快速导航。

### 个人主页

个人主页管理用户和配置。

- 注册、登录和用户昵称。
- 系统头像、上传头像和裁剪头像。
- API 配置，包括文本模型、图像模型和图像理解模型。
- 可切换使用系统默认模型或手动填写模型 API Key。
- 自动保存间隔设置。
- 用户技能等级设置。
- 项目历史查看和作品管理。

## 关键工作流

### AI 生成图案工作流

1. 用户选择主题、元素、作品形式、比例和配色。
2. 前端调用 `/api/generate-culture-image`。
3. 服务端根据配置构造传统文化图案提示词。
4. Ark 图像生成接口返回图像。
5. 图像进入第二步，用户可继续生成拼豆图纸。

### 上传图片再创作工作流

1. 用户上传本地图片。
2. 系统默认整张图为主体模板，并生成颜色占比。
3. 用户可修改主体区域。
4. 前端调用 `/api/identify-subject` 识别主体。
5. 用户可编辑结构化主体识别结果。
6. 前端调用 `/api/extract-theme-image` 进行文化风格再创作。
7. 再创作结果自动识别边缘以内区域作为主体。
8. 进入拼豆图纸生成。

### 拼豆图纸生成工作流

1. 读取再创作结果中的主体图像。
2. 按指定网格尺寸采样像素。
3. 映射到开源色表。
4. 限制颜色数量并合并近似颜色。
5. 清理外部背景和杂点。
6. 渲染带色号、行列号和网格线的图纸。
7. 统计颜色用量并生成导出资料。

## API 路由

| 路由 | 功能 |
|---|---|
| `/api/generate-culture-image` | 根据主题配置生成传统文化图案 |
| `/api/identify-subject` | 使用多模态文本模型识别图像主体，返回结构化 JSON |
| `/api/extract-theme-image` | 根据主体识别结果和颜色组成进行文化风格再创作 |
| `/api/generate-culture-text` | 根据图像、主体识别和材料数据生成文化说明 |
| `/api/generate-product-scene` | 生成产品场景预览 |
| `/api/chat` | 豆韵 AI 聊天式生图接口 |
| `/api/community-posts` | 云端社区作品读取和发布 |
| `/api/env-config` | 返回服务端模型配置状态 |
| `/api/matting` | 图像抠图辅助接口 |

## 关键技术

### 前端

- **Next.js 15 App Router**：页面、API 路由和构建框架。
- **React 19**：组件状态、交互流程和局部视图切换。
- **TypeScript**：主题、项目记录、颜色、图纸和社区数据类型。
- **Tailwind CSS 4**：界面布局、响应式设计和视觉样式。
- **PWA 支持**：通过 `next-pwa` 提供渐进式应用能力。
- **Vercel Analytics**：用于前端访问分析。

### 图像处理

- **Browser Canvas API**：读取图像、生成 mask、像素采样、绘制图纸和导出 PNG。
- **Sharp**：服务端压缩 data URL 图片，降低多模态接口输入体积。
- **主体 mask**：支持自动检测、整图选择、空选区、点击连通区域、笔刷增减和框选识别。
- **颜色占比分析**：对主体区域进行聚类，生成主要颜色和百分比。

### 颜色和像素算法

- **主导色采样**：每个格子按主导像素颜色映射到拼豆色。
- **Oklab/距离映射**：使用颜色距离寻找最接近的开源色表颜色。
- **颜色数量控制**：通过合并近似颜色而不是简单丢弃低频颜色，降低颜色失真。
- **外部背景识别**：从边界浅色区域开始做连通识别，标记为外部背景。
- **杂点处理**：平滑单格噪声，并可连接小孤岛区域。
- **色号显示**：每个非外部格子显示开源色号，便于实际摆豆。

### AI 能力

- **图像生成**：调用 Ark 图片生成模型。
- **主体识别**：使用与文化说明一致的多模态文本模型看图，返回严格 JSON。
- **文化再创作**：将用户确认后的主体识别、颜色组成和产品形式转为图像生成提示词。
- **文化说明生成**：读取图像、主体识别结果、材料颜色和产品形式，生成作品介绍。
- **提示词可编辑**：第二步和第四步都显示当前提示词，用户可以修改后重新生成。

### 数据存储

- **localStorage**：保存语言、用户配置、API 配置、AI 聊天历史等轻量数据。
- **IndexedDB**：保存项目历史，支持较大的图像 data URL 和多用户项目隔离。
- **云端社区接口**：用于发布和读取论坛作品。

## 环境变量

本地可创建 `.env.local`：

```bash
ARK_API_KEY=你的 Ark API Key
ARK_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
AI_TEXT_MODEL=doubao-seed-1-6-250615
AI_IMAGE_MODEL=doubao-seedream-4-0-250828
AI_VISION_MODEL=可选，当前主体识别默认复用文本多模态模型
```

说明：

- `/api/generate-culture-image` 使用图像生成模型。
- `/api/identify-subject` 当前复用文本多模态模型看图，避免独立视觉模型权限不一致导致第二步失败。
- `/api/generate-culture-text` 使用文本多模态模型生成文化说明。
- 个人主页可选择使用系统默认模型或手动填写模型信息。

## 典型使用方式

### 从传统主题开始

1. 进入创作页。
2. 选择传统主题和核心元素。
3. 设置作品形式、比例、网格和颜色数量。
4. 点击 AI 生成图案。
5. 生成拼豆图纸并导出资料。

### 从本地图片开始

1. 上传图片。
2. 默认整图作为主体模板。
3. 点击 AI 识别主体，必要时手动修改结果。
4. AI 再创作生成文化风格图案。
5. 生成拼豆图纸、文化说明和制作方案。

### 从论坛作品开始

1. 进入论坛页。
2. 浏览或搜索作品。
3. 打开作品详情。
4. 一键导入为自己的项目。
5. 继续修改主题、配色、网格和图纸。

## 色板数据

色板数据位于：

```text
src/app/colorSystemMapping.json
```

每个 hex 色值对应三类开源标注：

- `heritage`：传统色号。
- `palette`：配色编号。
- `sequence`：顺序编号。

这些字段只用于图纸展示和统计，不包含外部供应、购买或销售推荐语义。
